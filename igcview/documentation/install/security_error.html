IGCview - a security error has occurred!!!

You have attempted to load or save a file on your PC, without your browser having been configured with the appropriate permissions.

The fix depends on whether you have IGCview installed locally on your PC or whether you are accessing it over the internet:

If you want to access local files and you have IGCview also loaded locally, then you will need to add the directory containing igcview.html to an environment variable called CLASSPATH. A description of how to do this in contained with the local section of the IGCview installation notes.

If you are accessing IGCview via the internet but want to load or save log files held locally on your PC (i.e. your own log files), then you will need to define the IGCview web site in your browser as a trusted site. Instructions on how to do this are contained within the internet section of the IGCview installation notes.

I hope the above notes help. Frankly, accessing local files from within a web application such as IGCview is an area of little standardization between platforms and browsers. IGCview is written to use no platform-specific functionality, but configuration of security settings is something done very much at the system level, outside of the application itself.
